# AI活用アイデアシート Webアプリ

ウォーターフォール開発の上流工程でAIツール（NotebookLM、Gemini）を活用するためのアイデアシートを簡単に入力・管理できるWebアプリケーションです。

## 特徴

- **入力しやすいフォーム形式**: 表形式ではなく、直感的に入力できるフォーム形式
- **複数アイデアの管理**: 複数のアイデアを追加・削除・編集可能
- **構造化テキスト出力**: 入力内容を構造化されたテキスト形式で出力
- **JSON形式でのエクスポート**: プログラムで処理しやすいJSON形式でも出力可能
- **プレースホルダー対応**: `{プロジェクト名}`、`{システム名}`などのプレースホルダーを使用可能

## 技術スタック

- Next.js 14
- React 18
- TypeScript
- Tailwind CSS

## セットアップ

### 1. 依存関係のインストール

```bash
npm install
```

### 2. 開発サーバーの起動

```bash
npm run dev
```

ブラウザで [http://localhost:3000](http://localhost:3000) を開きます。

### 3. ビルド

```bash
npm run build
npm start
```

## 使い方

### 基本的な使い方

1. **アイデアの入力**
   - 各項目を入力してください（*は必須項目）
   - 複数のアイデアを追加する場合は「+ 新しいアイデアを追加」ボタンをクリック

2. **アイデアの管理**
   - 各アイデアは個別に編集・削除可能
   - 削除ボタンは2つ以上のアイデアがある場合に表示されます

3. **エクスポート**
   - 「エクスポート」セクションから出力形式を選択
   - 「クリップボードにコピー」または「ファイルをダウンロード」で出力

### 入力項目

#### 必須項目

- **開発工程**: 要求整理、要件定義、基本設計、その他PM業務から選択
- **ASIS：現状のタスク**: 現在の業務で困っていること
- **使用ツール**: NotebookLM または Gemini
- **TOBE：AI活用アイデア**: 具体的な指示内容（プレースホルダー使用可）
- **難易度**: 初級、中級、上級から選択
- **所要時間**: 目安の作業時間（例：15分）
- **期待効果**: 高、中から選択

#### 任意項目

- **準備資料**: 必要な資料の種類と形式
- **期待される成果物**: AIから得られる成果物の例
- **実施日**: 実践した日付
- **実施結果**: ○（うまくいった）、△（改善が必要）、×（うまくいかなかった）
- **カスタマイズ内容**: プロジェクトに合わせて調整した内容
- **備考・気づき**: 次回に活かせるポイント、改善点など

### プレースホルダーの使い方

TOBE：AI活用アイデアの入力欄で、以下のプレースホルダーを使用できます：

- `{プロジェクト名}`: プロジェクト名に置き換え
- `{システム名}`: システム名に置き換え
- `{日付範囲}`: 日付範囲に置き換え

例：
```
全ての議事録PDFをソースとして読み込ませ、「{プロジェクト名}の顧客が『課題』『問題』『困っている』と発言した箇所を全て抜き出し、トピック別に分類して。各課題の背景と影響範囲も含めて整理して」と指示する。
```

## 出力形式

### 構造化テキスト形式

Markdown形式で構造化されたテキストとして出力されます。AIチャットやドキュメント作成にそのまま使用できます。

### JSON形式

プログラムで処理しやすいJSON形式で出力されます。データの構造を確認したい場合や、他のシステムと連携する場合に便利です。

2. Vercelでプロジェクトをインポート

- Google Cloud Run
- Azure Static Web Apps
