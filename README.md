# AI活用アイデアシート Webアプリ

ウォーターフォール開発の上流工程でAIツール（NotebookLM、Gemini）を活用するためのアイデアシートを簡単に入力・管理できるWebアプリケーションです。

## 特徴

- **入力しやすいフォーム形式**: 表形式ではなく、直感的に入力できるフォーム形式
- **複数アイデアの管理**: 複数のアイデアを追加・削除・編集可能
- **構造化テキスト出力**: 入力内容を構造化されたテキスト形式で出力
- **JSON形式でのエクスポート**: プログラムで処理しやすいJSON形式でも出力可能
- **プレースホルダー対応**: `{プロジェクト名}`、`{システム名}`などのプレースホルダーを使用可能

## 技術スタック

- Next.js 14
- React 18
- TypeScript
- Tailwind CSS

## セットアップ

### 1. 依存関係のインストール

```bash
npm install
```

### 2. 開発サーバーの起動

```bash
npm run dev
```

ブラウザで [http://localhost:3000](http://localhost:3000) を開きます。

### 3. ビルド

```bash
npm run build
npm start
```

## 使い方

### 基本的な使い方

1. **アイデアの入力**
   - 各項目を入力してください（*は必須項目）
   - 複数のアイデアを追加する場合は「+ 新しいアイデアを追加」ボタンをクリック

2. **アイデアの管理**
   - 各アイデアは個別に編集・削除可能
   - 削除ボタンは2つ以上のアイデアがある場合に表示されます

3. **エクスポート**
   - 「エクスポート」セクションから出力形式を選択
   - 「クリップボードにコピー」または「ファイルをダウンロード」で出力

### 入力項目

#### 必須項目

- **開発工程**: 要求整理、要件定義、基本設計、その他PM業務から選択
- **ASIS：現状のタスク**: 現在の業務で困っていること
- **使用ツール**: NotebookLM または Gemini
- **TOBE：AI活用アイデア**: 具体的な指示内容（プレースホルダー使用可）
- **難易度**: 初級、中級、上級から選択
- **所要時間**: 目安の作業時間（例：15分）
- **期待効果**: 高、中から選択

#### 任意項目

- **準備資料**: 必要な資料の種類と形式
- **期待される成果物**: AIから得られる成果物の例
- **実施日**: 実践した日付
- **実施結果**: ○（うまくいった）、△（改善が必要）、×（うまくいかなかった）
- **カスタマイズ内容**: プロジェクトに合わせて調整した内容
- **備考・気づき**: 次回に活かせるポイント、改善点など

### プレースホルダーの使い方

TOBE：AI活用アイデアの入力欄で、以下のプレースホルダーを使用できます：

- `{プロジェクト名}`: プロジェクト名に置き換え
- `{システム名}`: システム名に置き換え
- `{日付範囲}`: 日付範囲に置き換え

例：
```
全ての議事録PDFをソースとして読み込ませ、「{プロジェクト名}の顧客が『課題』『問題』『困っている』と発言した箇所を全て抜き出し、トピック別に分類して。各課題の背景と影響範囲も含めて整理して」と指示する。
```

## 出力形式

### 構造化テキスト形式

Markdown形式で構造化されたテキストとして出力されます。AIチャットやドキュメント作成にそのまま使用できます。

### JSON形式

プログラムで処理しやすいJSON形式で出力されます。データの構造を確認したい場合や、他のシステムと連携する場合に便利です。

## セキュリティとプライバシー

> 📋 詳細なセキュリティ情報は [SECURITY.md](./SECURITY.md) をご覧ください。

### データは一切保存されません

このアプリケーションは、**入力されたデータを一切保存しません**。機密情報を含む業務課題を安全に入力できます。

#### 技術的な証明

1. **クライアントサイドのみで動作**
   - すべての処理はブラウザ内で完結します
   - サーバーサイドのAPIやデータベースは使用していません
   - Next.jsの静的サイト生成（SSG）を使用しており、サーバーサイドの処理はありません

2. **データ保存の不使用**
   - ローカルストレージ（localStorage）: 使用していません
   - セッションストレージ（sessionStorage）: 使用していません
   - Cookie: 使用していません
   - IndexedDB: 使用していません
   - サーバーへのデータ送信: 行っていません

3. **データの管理方法**
   - すべてのデータはReactの`useState`フックで管理されています
   - データはブラウザのメモリ（RAM）にのみ一時的に存在します
   - ページを閉じると、すべてのデータは完全に消去されます

4. **エクスポート機能**
   - エクスポート機能もブラウザ内で完結します
   - ファイルはお使いのデバイスにのみダウンロードされます
   - サーバーにアップロードされることはありません

#### ソースコードの確認

すべてのソースコードは[GitHubで公開](https://github.com/Michey0495/ai-idea-worksheet)されています。以下の点を確認できます：

- `app/page.tsx`: データは`useState`で管理されており、外部ストレージは使用していません
- `components/ExportSection.tsx`: エクスポート機能はブラウザ内で完結し、ファイルダウンロードのみです
- APIルート（`app/api/`）: 存在しません
- データベース接続コード: 存在しません
- 外部API呼び出し: 存在しません

#### 機密情報の安全な使用

- 機密情報を含む業務課題を入力しても安全です
- データは一切保存されず、ページを閉じると完全に消去されます
- エクスポートしたファイルは、お使いのデバイスにのみ保存されます
- ファイルの管理にはご注意ください

#### 検証方法

1. **ブラウザの開発者ツールで確認**
   - F12キーで開発者ツールを開く
   - Applicationタブ > Storage を確認
   - Local Storage、Session Storage、Cookies が空であることを確認

2. **ネットワークタブで確認**
   - Networkタブを開く
   - データを入力・エクスポートしても、サーバーへのリクエストがないことを確認

3. **ソースコードで確認**
   - GitHubリポジトリのコードを確認
   - データ保存やAPI呼び出しのコードがないことを確認
